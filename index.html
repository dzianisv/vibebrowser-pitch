Why In-Browser vs. Cloud-Hosted Agents?

Cloud agents (like OpenAI's browser agent) face critical limitations:

  • Geo/IP blocking: Cloud IPs are blocked. Residential IPs required for real-world sites.
  • Limited context: No access to user's browsing history, saved credentials, or sessions.
  • CAPTCHA & anti-bot: Cloud browsers get flagged and blocked by anti-bot systems.
  • Modern auth: Passkey authentication requires local deviceâ€”impossible in cloud.
  • Privacy: User data stays local, not sent to cloud infrastructure.

Technical Architecture & Delivery

Extension (MVP)

Chromium extension with DOM control. Lowest friction.

Tech: Content scripts, proprietary automation framework, GPT-5-mini.

Custom Browser

Chromium fork for deeper context access beyond extension APIs.

Tech: Native Gemini Nano integration, LangGraph ReAct workflows.

Cloud Agent

Headless Chromium for long-running automations.

Tech: Playwright, containerized sessions, audit logs.
